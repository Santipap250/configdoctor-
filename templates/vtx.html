💻 CLI Commands Reference — ทุก Protocol
SmartAudio (Betaflight CLI)
ใช้กับ VTX ที่รองรับ TBS SmartAudio — ตั้งค่าผ่าน CLI หรือ OSD → Video Transmitter Settings
# ตัวอย่าง: Raceband Ch1 (5658 MHz), 200mW, SmartAudio
set vtx_band = 5 # 1=A, 2=B, 3=E, 4=F, 5=R, 6=D
set vtx_channel = 1 # Ch 1–8
set vtx_power = 2 # index: 1=25mW, 2=200mW, 3=500mW ...
set vtx_low_power_disarm = ON # ลดกำลังส่งขณะ disarm (แนะนำ)
save
IRC Tramp (Frequency Direct)
ส่ง command โดยตรงด้วยความถี่ MHz — ยืดหยุ่นกว่า SmartAudio
# IRC Tramp: ตั้งตรงๆ ด้วย freq + power
# vtx <frequency_mhz> <power_mw>
vtx 5658 200 # R1 @ 5658 MHz, 200mW
# หรือตั้งแบบ band/channel:
vtxtable band 5 # band index
vtxtable channel 1

📡 ทำความเข้าใจ VTX
📡
VTX คืออะไร?
VTX (Video Transmitter) คืออุปกรณ์ส่งสัญญาณวิดีโอจากโดรนไปยัง Goggle/Monitor ในความถี่ 5.8 GHz แบบ analog หรือ digital (DJI, Walksnail, HDZero)
ความถี่ 5.8 GHz แบ่งเป็น Band (กลุ่ม) แต่ละ Band มี 8 Channel ในแต่ละ Channel คือ ความถี่ MHz ที่ต่างกัน
🌊
Band vs Channel คืออะไร?
Band = กลุ่มความถี่ (เหมือนสถานีวิทยุ เช่น FM, AM)
Channel = ความถี่ MHz เฉพาะในแต่ละ Band

เช่น Raceband Ch1 = 5658 MHz
หรือ Band A Ch1 = 5865 MHz
pilot 2 คนที่ใช้ channel ใกล้กัน (น้อยกว่า 20 MHz) จะ interfere กัน ทำให้ภาพแตกหรือหายไปได้ในระหว่างบิน
🏁
ทำไม Raceband ถึงดีที่สุด?
Raceband ถูกออกแบบมาเฉพาะสำหรับการแข่งขัน FPV — ช่องทั้ง 8 เว้นระยะห่างกัน 37 MHz เท่าๆ กัน ซึ่งมากกว่า threshold interference (20 MHz)
  • ✅ สามารถบินพร้อมกันได้ 8 pilot โดยไม่ interference กัน
  • ✅ เป็นมาตรฐาน MultiGP / FAI FPV ทั่วโลก
  • ✅ ช่อง R1–R4 อยู่นอกโซน WiFi 5.8GHz
  • ⚠️ R7 (5880) overlap กับ WiFi Band ที่ใช้บ่อย
📶
WiFi Interference คืออะไร?
WiFi 5GHz (802.11ac/n) ใช้ความถี่ 5725–5875 MHz ซึ่ง overlap กับหลาย Channel ของ FPV VTX
ช่อง F1–F8 ทับกับ WiFi ทั้งหมด! Band B / Band A บางช่องก็ทับด้วย หาก Router หรือ Drone มี WiFi ใกล้ — ภาพจะรบกวน
แนะนำ: ใช้ Raceband R1–R4 เพื่อหลีกเลี่ยง WiFi

💡 Tips & Best Practice
💡 Tips สำหรับ Race Day & การใช้งาน VTX
  • 🏁
    Race Day: ประสานงาน channel ก่อน arm เสมอ ใช้ Race Planner ในแท็บด้านบนเพื่อหลีกเลี่ยง conflict กับ pilot คนอื่น
  • 🌡️
    VTX ร้อน: VTX ที่กำลังสูงโดยไม่ได้บินจะร้อนมาก — ตั้ง set vtx_low_power_disarm = ON เพื่อลดกำลังส่งขณะ disarm อัตโนมัติ
  • 🏠
    Indoor / Proximity: ใช้ไม่เกิน 25–100 mW ในอาคาร สัญญาณสะท้อนผนังรบกวนมากกว่ากำลังส่งต่ำ
  • 📶
    หลีกเลี่ยง WiFi: Raceband R1–R4 (5658–5769 MHz) อยู่นอกโซน WiFi ถ้าสนามมี Router เยอะ เลือกช่องนี้ก่อน
  • 🎥
    DJI / Digital VTX: Digital VTX (DJI O3, Walksnail, HDZero) ใช้ protocol และ channel system ต่างกัน ไม่ต้องตั้ง vtx_band ใน Betaflight เพราะจัดการเองผ่าน app
  • ⚙️
    vtx_low_power_disarm: แนะนำเสมอสำหรับ pilot ทุกคน ป้องกัน VTX ร้อน และลด RF noise ขณะ setup บนสนาม
  • 📡
    Antenna ทิศทาง: Antenna ของ Goggle ควรหันเข้าหาทิศทางบิน Circularly Polarized (Clover/AXII) ดีกว่า Dipole ในสนามแข่ง
  • 🔋
    Pitmode: VTX บางรุ่นมี Pit Mode (กำลังส่งต่ำมาก ~1mW) — ใช้ขณะนั่ง paddock เพื่อไม่รบกวนคนบินอยู่
